# UPC Image Selector

A production-style **Java 21 / Spring Boot 3** application that automatically
selects the best front-of-pack product image for each UPC using fully **local**
heuristics — no cloud APIs, no paid services, no external AI models.

---

## How it works

```
ImagesLink.txt
     │
     ▼  (concurrent HTTP download)
downloaded_images/
     │
     ▼  (per-image scoring)
     │   • Resolution   (0-20 pts)  — more pixels = more detail
     │   • Sharpness    (0-25 pts)  — Laplacian variance
     │   • Brightness   (0-15 pts)  — ideal mean ≈ 150/255
     │   • Contrast     (0-15 pts)  — std-dev of brightness
     │   • Background   (0-15 pts)  — white / plain border region
     │   • Centering    (0-10 pts)  — foreground COM near image centre
     │   • Type bonus   (0-2  pts)  — type 1 > 70 > 74 > 21 (weak)
     ▼
selected_front_images/          ← best image per UPC copied here
selected_front_images.txt       ← plain list of selected filenames
selected_front_images.csv       ← full score breakdown per UPC
results/processing_results.json ← persisted state (survives restarts)
```

## Quick start

### 1. Clone and build

```bash
git clone <repo-url>
cd upc-image-selector
mvn clean package -DskipTests
```

### 2. Populate `ImagesLink.txt`

Place one image URL per line.  Each URL must contain a filename in the form
`{14-digit-UPC}_{imageType}.jpg`, for example:

```
https://cdn.example.com/products/00012345678901_1.jpg
https://cdn.example.com/products/00012345678901_70.jpg
https://cdn.example.com/products/00012345678902_74.jpg
```

Comments (`#`) and blank lines are ignored.

### 3. Run

```bash
java -jar target/upc-image-selector-1.0.0.jar
```

The app starts on **http://localhost:8080**.

### 4. Trigger processing

Option A – UI:  open http://localhost:8080 and click **Start Processing**.

Option B – API:

```bash
# start processing
curl -X POST http://localhost:8080/api/process

# poll progress
curl http://localhost:8080/api/status
```

### 5. Review & override

Open http://localhost:8080/review to see a thumbnail grid for every UPC.
Click **Select** on any candidate to override the automated choice.

## Configuration (`application.yml`)

```yaml
app:
  images-link-file: ImagesLink.txt      # path to URL list
  download-dir: downloaded_images       # where images are saved
  selected-dir: selected_front_images   # where selected images are copied
  results-dir: results                  # JSON persistence directory
  download-threads: 10                  # concurrent download threads
  download-timeout-seconds: 30
  read-timeout-seconds: 60
  max-image-size-bytes: 52428800        # 50 MB limit per image
  scoring:
    working-size: 600         # downsample to this before scoring (speed)
    border-fraction: 0.15     # fraction of image used as border region
    sharpness-scale: 500.0    # Laplacian variance normalisation factor
    ideal-brightness: 150.0   # 0-255 target mean brightness
    ideal-contrast-low: 40.0  # std-dev lower bound for full contrast score
    ideal-contrast-high: 80.0 # std-dev upper bound for full contrast score
```

## REST API

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/process` | Start pipeline (async, poll `/api/status`) |
| `GET`  | `/api/status` | Processing progress and state |
| `GET`  | `/api/results` | All UPC results with score breakdowns |
| `GET`  | `/api/results/{upc}` | Single UPC result |
| `POST` | `/api/results/{upc}/override` | Override selected image |
| `GET`  | `/api/export/txt` | Download `selected_front_images.txt` |
| `GET`  | `/api/export/csv` | Download `selected_front_images.csv` |

### Override example

```bash
curl -X POST http://localhost:8080/api/results/00012345678901/override \
     -H "Content-Type: application/json" \
     -d '{"filename":"00012345678901_70.jpg"}'
```

## Scoring algorithm detail

The scorer never calls any external API.  All analysis is pure Java pixel math.

### Sharpness (Laplacian variance)

```
kernel = [[0, 1, 0],
          [1,-4, 1],
          [0, 1, 0]]

For each interior pixel (sampled at stride for speed):
  response = kernel convolved at pixel

variance(responses) → normalised to [0, 25]
```

High variance means the image has strong local edges → in focus.

### Background plainness

The outer 15 % of the image is sampled.
Each border pixel is converted to HSB; the mean **saturation** and **brightness**
(in the HSB sense) are measured:

```
score = (1 - mean_saturation) * 0.5 + mean_brightness * 0.5
```

Low saturation + high brightness ≈ white / plain background.

### Subject centering

Background brightness is estimated from image corners.
Pixels significantly darker than the background are treated as foreground.
The foreground centre-of-mass is computed and its distance from the image
centre is normalised to [0, 10].
